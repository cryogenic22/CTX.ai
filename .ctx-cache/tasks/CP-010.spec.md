# CP-010 — Symbol → IREntity emitter

**Status**: shipped
**Backlog**: CP-010 (deps CP-003–CP-009)
**Effort**: 1d

## Acceptance
- One entity per symbol; all required fields populated; `centrality_prior` placeholder 0.0 for now.

## Design
New `ctxpack/core/code/emitter.py`:

```python
def emit_irentities(
    result: ParseResult,
    file_path: str | Path,
    *,
    include_body: bool = True,
) -> list[IREntity]:
```

For each symbol (deterministic file order from extract_symbols), emit one IREntity with:
- `name` = qualified name (CP-009).
- `sources` = [IRSource(file=normalised, line_start, line_end)].
- `fields` (IRField, layer=RULES, confidence=1.0):
  - `kind` (always)
  - `signature` (single line — `def foo(...) -> Bar:` or `class Foo(BaseModel):`)
  - `docstring` (first child of body if string literal, else "")
  - `body` (full source text — only when include_body=True)
  - `decorators` (JSON-encoded list of dotted names) when non-empty
  - `decorators_full` (JSON: list of {name, args, kwargs, line}) when non-empty
  - `http_method` / `http_path` if extract_route returns a RouteInfo
  - `dependencies` (JSON list of {parameter, target}) if extract_dependencies returns any
  - `pydantic_fields` (JSON list of {name, type, default}) if pydantic_fields returns any
  - `centrality_prior` = `"0.0"` placeholder (CP-019/20 populates)
- `layer` = RULES, `confidence` = 1.0.

Body cap: 16K BPE per symbol via `count_bpe`. If a body exceeds the cap, store first 16K BPE + a `[truncated N bytes]` marker. Pinned via `count_bpe` from CP-002.5.

## Tests
- read_user emits an entity with http_method/http_path/dependencies set.
- User (Pydantic) emits an entity with pydantic_fields, NOT http_method.
- Plain function gets no decorator-derived fields.
- Class attribute emits an entity with kind="class_attribute", no body slicing weirdness.
- Body field is omitted when include_body=False.
- Body truncation kicks in for a >16K BPE source (synthetic huge file).
- centrality_prior is "0.0" on every entity.
- Each entity's sources[0].line_start/line_end match the symbol's range.
