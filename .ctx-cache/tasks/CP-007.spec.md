# CP-007 — FastAPI `Depends(...)` extraction

**Status**: shipped
**Backlog**: CP-007 (Week 1, deps CP-006)
**Effort**: 0.5d

## Acceptance
- A route with `Depends(get_db)` lists `get_db` in dependencies.
- Nested Depends are flagged but only depth-1 resolved.

## Design
Add to `ctxpack.core.code.metadata`:
```python
@dataclass(frozen=True)
class Dependency:
    parameter: str   # "db"
    target: str      # "get_db" (source text of the call arg)

def extract_dependencies(result: ParseResult, symbol: Symbol) -> list[Dependency]:
```

Find the function_definition node at symbol.byte_start via `tree.root_node.descendant_for_byte_range`. Walk parameters → default_parameter / typed_default_parameter → if `.value` is a `call` whose function is exactly `Depends`, capture `(parameter_name, first_positional_arg_source)`.

Non-function symbols return `[]`.

## Tests
- read_user / create_user in py_fastapi_min: each has `Depends(get_db)` → `Dependency("db", "get_db")`.
- Function without Depends → empty list.
- Class symbol → empty list.
- typed_default_parameter shape covered (`db: Session = Depends(get_db)`).
