# CP-008 — Pydantic BaseModel detection

**Status**: shipped
**Backlog**: CP-008 (Week 1, deps CP-004)

## Acceptance
- Each Pydantic BaseModel subclass in fixture emits one entity with `pydantic_fields=[{name, type}]`.

## Design (revised for CP-008 v0)
At v0 the cleanest contract is:
- `is_pydantic_model(result, symbol) -> bool` — True if `symbol.kind == CLASS` and at least one base class identifier ends in `BaseModel` (handles `pydantic.BaseModel`, `from pydantic import BaseModel`, `class Foo(SomeOtherBaseModel)`).
- `pydantic_fields(result, symbol) -> list[PydanticField]` — list of typed class attributes captured from the class body. Each field has `name`, `type_source`, `default_source` (optional).

CP-010 reads these to emit `pydantic_fields` IRField.

## Tests
- `User(BaseModel)` in py_fastapi_min → is_pydantic_model True.
- `User` has 3 pydantic_fields (id:int, name:str, email:str), no defaults.
- `Widget` (plain class) → is_pydantic_model False, pydantic_fields empty.
- Non-class symbol (function) → is_pydantic_model False.
