# CP-019 + CP-020 — Weighted PageRank + centrality_prior population

**Status**: shipped
**Backlog**: CP-019, CP-020 (Week 3, deps CP-014)
**Effort**: 1.5d combined.

## Acceptance (CP-019)
- Output scores normalised to sum to 1.
- Test edges weight 3× call edges.
- Tie-breaking on identical scores is deterministic (alphabetical).

## Acceptance (CP-020)
- After pack, each entity has populated `centrality_prior` ∈ [0,1].

## Design
New `ctxpack/core/code/ranker.py`:
```python
def compute_pagerank(
    nodes: set[str],
    edges: Iterable[CallEdge],
    *,
    test_edges_weight: float = 3.0,
    export_pin_weight: float = 1.5,
    damping: float = 0.85,
    iterations: int = 30,
    test_node_predicate: Callable[[str], bool] | None = None,
) -> dict[str, float]:
```

Power-iteration PageRank, deterministic. `test_node_predicate` defaults to "caller starts with `tests/` or `tests\\`" so test-call edges get the 3× weight only when the caller is a test.

For CP-020: a `populate_centrality_prior(entities, scores)` helper that updates each entity's `centrality_prior` IRField in-place.

## Tests
- Mini graph: 3 nodes, known PageRank, scores sum to 1 within tolerance.
- Test edges weight 3× call edges.
- Determinism: two calls return identical dicts (alphabetical tie-break).
- Isolated node (no incoming edges) still gets non-zero score (damping floor).
- populate_centrality_prior updates the field correctly.
