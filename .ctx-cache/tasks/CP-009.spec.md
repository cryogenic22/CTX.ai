# CP-009 — Symbol naming scheme + overload disambiguation

**Status**: shipped
**Backlog**: CP-009 (deps CP-004)
**Effort**: 0.5d

## Acceptance
- Two functions with same name in different scopes get distinct stable names.
- Names round-trip through serialization.

## Design
New module `ctxpack/core/code/naming.py`:
- `qualified_name(file_path, symbol) -> str` — `"path/forward/slashed.py::symbol.name"`.
- `qualified_names_for_module(file_path, symbols) -> list[tuple[Symbol, str]]` — same, but collisions get `#1`, `#2`, ... suffixes in file order.

Symbol stays unchanged. Qualified naming is layered on top so callers that don't need it pay nothing.

## Tests
- Bare function: `src/foo.py::main`.
- Method: `src/foo.py::Widget.tick`.
- Two `@overload` defs of same name in same file → first is `name`, second is `name#1`.
- Class with method of same name as a top-level function in same file: distinct names (different scopes).
- Round-trip through JSON dump+load preserves names.
- Windows-style path on input → forward slashes on output (`src\\foo.py` → `src/foo.py`).
