# CP-010.3 — Catalog-row renderer (soft cap)

**Status**: shipped
**Backlog**: CP-010.3 (deps CP-010, CP-002.5)
**Effort**: 0.5d

## Acceptance
- Short signature passes through.
- 200-BPE Python signature truncates with `…`.
- TSX generic-heavy signature truncates at a generic boundary, not mid-identifier.

## Design
New `ctxpack/core/code/catalog.py`:
```python
def render_catalog_row(entity: IREntity, *, cap: int = 120) -> str:
    """name + kind + signature + docstring-first-line, soft-capped to `cap` BPE."""
```
Truncation strategy: if (signature length is what blows the cap) truncate signature; prefer to break at one of `,`, `]`, `)`, `>`, `[`, ` ` (descending priority); append `…`. Name is never truncated.

## Tests
- Short row returns verbatim.
- Long Python signature truncates with `…`.
- Heavy generics truncate at a boundary char.
- Row never exceeds cap + small overhead (single ellipsis).
- Name preserved exactly even when truncating other components.
