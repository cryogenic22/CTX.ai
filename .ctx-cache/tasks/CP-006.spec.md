# CP-006 — FastAPI route detection

**Status**: shipped
**Backlog**: CP-006 (Week 1, deps CP-005)
**Effort**: 0.5d

## Acceptance
- `@app.*` / `@router.*` HTTP-method decorators on a symbol → `RouteInfo(http_method, http_path)`.
- Non-route decorators don't trip the detector.

## Design
Pure decorator interpretation; no AST access. Lives in `ctxpack/core/code/metadata.py`.

```python
HTTP_METHODS = frozenset({"get","post","put","patch","delete","head","options","trace"})

@dataclass(frozen=True)
class RouteInfo:
    http_method: str  # uppercased
    http_path: str    # raw source text incl quotes if string literal

def extract_route(symbol: Symbol) -> RouteInfo | None:
    """Return route info if the symbol has a FastAPI-style route decorator."""
```

Match: decorator name is `<router>.<method>` where `router` is anything (covers `app.get`, `router.get`, `v1_router.post`, etc.) AND `<method>` is in HTTP_METHODS. First positional arg is the path.

## Tests
- Plain `@app.get("/foo")` → RouteInfo("GET", '"/foo"').
- `@router.post(...)` → POST.
- All HTTP methods covered.
- `@app.get` without call → still a route, path is None.
- Non-route decorator (`@cache`, `@property`) → None.
- Function with multiple decorators, one is the route, returns that route.
- Symbol with no decorators → None.
