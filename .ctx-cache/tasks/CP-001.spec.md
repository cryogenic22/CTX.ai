## Design

Two new package roots:

- `ctxpack.core.code` ‚Äî sits alongside `ctxpack.core.packer` (existing
  prose-domain producer). Code-packer is a sibling producer per
  RFC ¬ß12 ("one IR, two producers, one MCP surface"). v0 has nothing
  but `__init__.py` until CP-002 lands the parser.
- `tests/code/` ‚Äî mirrors the producer package structure under
  tests. Fixture repos live at `tests/code/fixtures/<name>/` so they
  can be referenced via `Path(__file__).parent / "fixtures" / ...`.

Fixture shape (Python+FastAPI minimal):

- `app.py` ‚Äî one FastAPI app, two routes (`@app.get`, `@app.post`), one
  with `Depends(...)`. Enough to seed CP-006/007 fixture coverage
  without committing to every edge case now.
- `models.py` ‚Äî one Pydantic `BaseModel` subclass with a couple of
  typed fields. Seeds CP-008.
- `deps.py` ‚Äî one dependency function (`get_db`) the route in `app.py`
  references via `Depends`. Splitting it into its own file matters
  for CP-014 call-graph testing later.

`slow` marker decision: register in `pyproject.toml` `[tool.pytest.ini_options]`
under a new `markers = [...]` list. Annotate the two known-slow modules
at class level (one decorator per class) rather than per-function ‚Äî both
modules invoke `analyze_codebase(SCRIPTIVA_ROOT)` against a real
156K-LOC codebase, so all tests in them are slow.

## Failure modes red-teamed in step [5]

- Test discovery picks up fixture `.py` files as test modules (Python
  files named `app.py` / `models.py` won't auto-collect by default ‚Äî
  pytest only collects `test_*.py` / `*_test.py` ‚Äî but worth a sanity
  check).
- `tests/code/__init__.py` missing causes pytest to not see the new
  tests under conftest-less rootdir discovery.
- `slow` marker not registered ‚Üí `PytestUnknownMarkWarning` becomes
  an error if `filterwarnings` is strict.
- `.gitignore` exception ordering ‚Äî `!.ctx-cache/tasks/` must come
  AFTER `.ctx-cache/*` to take effect.
